# Guide de Contribution

Merci de l'intérêt que vous portez au projet **testcreatespring** ! Voici comment vous pouvez contribuer.

## Comment Contribuer ?

### 1. Signaler un Bug
Si vous trouvez un bug, veuillez utiliser le template de rapport de bug dans les issues GitHub. Soyez le plus précis possible.

### 2. Proposer une Fonctionnalité
Les suggestions sont les bienvenues ! Créez une issue "Feature Request" pour en discuter.

### 3. Soumettre une Pull Request
1. Forkez le projet.
2. Créez une branche pour votre fonctionnalité (`git checkout -b feature/AmazingFeature`).
3. Commitez vos changements (`git commit -m 'Add some AmazingFeature'`).
4. Pushez vers la branche (`git push origin feature/AmazingFeature`).
5. Ouvrez une Pull Request.

## Standards de Code
- Suivez les conventions de nommage Java standard.
- Assurez-vous que le code est documenté si nécessaire.
- Vérifiez que les tests passent avant de soumettre.

## Processus de Pull Request
Toutes les PR doivent être liées à une issue existante. Elles seront revues avant d'être mergées.

Merci encore pour votre aide !
